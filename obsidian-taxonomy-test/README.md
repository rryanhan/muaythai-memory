# Muay Thai Taxonomy Test

This vault tests a simple Obsidian structure for the app taxonomy.

## Structure

- `4 - Index`: practice context pages such as [[Pad Work]], [[Sparring Drill]], and [[Clinch]].
- `3 - Tags`: Training Tags, Custom Tags, and Status pages.
- `6 - Drills`: the actual drill notes.

## Current Model

```txt
Training Contexts + Training Tags + Custom Tags + Status -> Drill
```

In Obsidian, every drill links directly to its contexts and tags. The graph should therefore cluster around practice contexts and frequently reused tags without forcing a strict hierarchy.

## Things To Test

- Do practice context nodes feel like useful anchors?
- Do broad and specific Training Tags work together, or does the graph get noisy?
- Should Status nodes be visible in the graph, or should they become app-only visual states later?
- Are Custom Tags useful, or do they create clutter quickly?
- Is `6 - Drills` the right place/name for saved drill entries?
