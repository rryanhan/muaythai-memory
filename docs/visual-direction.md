# Visual Direction

## Theme

The first visual direction is a light training-lab interface.

The app should feel:

- focused
- technical
- quiet
- sharp
- practical
- graph-friendly

It should not feel like:

- a generic fitness tracker
- a wellness journal
- a flashy sports app
- a textbook martial arts database
- a generic SaaS dashboard

## Fonts

Primary font:

- IBM Plex Sans

Optional mono font:

- IBM Plex Mono

IBM Plex Sans gives the app a slightly technical, lab-like tone while staying readable for notes, drill lists, and graph labels.

IBM Plex Mono should be used sparingly for metadata, compact labels, or technical UI details.

## Graph Styling

Default graph:

- Training Method nodes are largest and graphite.
- Drill nodes are smaller and muted gray.
- Tag nodes are teal when enabled.
- Custom Tag nodes are amber when enabled.
- Active search or active filters use red-orange.

The graph should start clean and method-first. Optional layers should add detail only when the user asks for it.

## UI Shape

Use restrained, utilitarian shapes:

- 6-8px border radius.
- Compact chips.
- Icon-first buttons when possible.
- Clean panels and sheets with borders.
- No decorative gradients.
- No large marketing-style cards.

## TasteSkill Usage

TasteSkill can be used as a design-quality helper for the local wireframe and later frontend prototype.

Use it as a rubric for stronger layout, spacing, hierarchy, and non-generic visual direction. It should not override the product decisions already made in the docs.

The first wireframe should remain low-fidelity enough to evaluate UX, but it should still follow this visual direction.
